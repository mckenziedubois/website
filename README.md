# website

This repo contains source code for mckenziedubois.com. It's a simple static site (HTML, CSS, and vanilla JS) with a small Python helper used to pull images/metadata from Cloudinary and generate CSV files consumed by the front-end.

Below you'll find a short guide to the project layout, key files, and common development tasks.

## Top-level folders & important files

- `index.html` — Home page.
- `posts/` — Standalone pages and generated CSV data (now being reorganized).
	- `image_metadata.csv` — Image metadata (url, orientation, country, pinurl, slug, alt, caption, exclude, header, order). Generated by the Python script.
- `guides/` — Guide templates and metadata
	- `guides/posts.json` — Guide metadata (title, slug, date, image, google_maps_link, etc.). The JS reads this to render guides.
	- `guides/<slug>/index.html` — Clean URL templates for each published guide (e.g., `/guides/hanoi/`, `/guides/hcmcity/`). Each template renders markdown and builds a Swiper gallery.
	- `guides/published/` — Markdown content for published guides (one `.md` file per guide slug). 
	- `guides/drafts/` — Local-only/unpublished drafts. These files may be git-ignored and are only read client-side when present.
	- `guides/post.html` — Legacy single-guide template (kept for backward compatibility).
	- `guides/blog.html` — Blog/guides home page listing all published guides.
- `partials/` — Reusable HTML fragments (header, footer, masthead, nav, scripts). These are injected at runtime by `js/load-partials.js`.
- `css/` — Stylesheets
	- `css/styles.css` — Primary theme / base styles.
	- `css/custom.css` — Local overrides and small component styles.
- `js/` — Front-end scripts
	- `js/load-partials.js` — Partial injection and masthead/nav helpers.
	- `js/csv-utils.js` — Shared CSV loader used across pages.
	- `js/gallery-filter.js` — Gallery logic: CSV loading, filtering, pagination, Cloudinary helpers.
	- other utilities: `config.js`, `gallery-filter.js`, `gallery-*` assets, etc.
- `python/` — Python helper scripts
	- `python/pull_cloudinary_images_posts.py` — Script to page through Cloudinary Search API, normalize `secure_url`, and write `pages/image_metadata.csv` and `pages/pinterest_metadata.csv`.

Other files:
- `ads.txt`, `CNAME` — deployment/hosting files.

## Guides / Content flow

To add a new guide:

1. **Create markdown content** — Add `guides/published/<slug>.md` with the guide text (or `guides/drafts/<slug>.md` for unpublished drafts).
2. **Add metadata** — Update `guides/posts.json` with: `title`, `slug`, `date`, `subtitle`, `image`, `tags`, `google_maps_link`, `airbnbs` (optional), and `overlay` (optional).
3. **Create URL template** — Copy an existing guide folder (e.g., `guides/hanoi/`) to `guides/<slug>/` and keep the `index.html` inside. The template automatically extracts the slug from the URL path.
4. **Add gallery images** (optional) — Upload images to Cloudinary in folder `gallery/<country>/<slug>/`, then run `python3 python/pull_cloudinary_images_posts.py` to regenerate the CSV.
5. **Set header image** (optional) — Mark a row's `header` column as `true` in Cloudinary context for the guide's slug, then rerun the Python script.

Clean URLs: Guides are accessed at `/guides/<slug>` (e.g., `/guides/hanoi/`, `/guides/hcmcity/`) via GitHub Pages' folder/index.html pattern.

## Common tasks

- Regenerate image CSVs: `python3 python/pull_cloudinary_images_posts.py`
- Add/edit guide: 
  - Create/edit `guides/published/<slug>.md` (or `guides/drafts/<slug>.md` for drafts)
  - Update `guides/posts.json` with metadata
  - Create `guides/<slug>/index.html` folder template (copy from existing guide)
